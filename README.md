Pizza Sales Data Analysis (SQL)

📌 Project Overview:
This project focuses on analyzing an entire year's worth of sales data from a pizza restaurant. Using structured SQL queries, I have extracted actionable business insights regarding sales performance, customer behavior, and product trends.

🛠️ Tech Stack & Concepts Used
- Database Platform: SQL
- Core Concepts: Joins, Aggregations, Subqueries, Window Functions, and Common Table Expressions (CTEs).

🚀 The 13 Business Questions Solved

📊 Basic Insights
- 01_total_orders: Retrieve the total number of orders placed.
- 02_total_revenue: Calculate the total revenue generated from pizza sales.
- 03_highest_priced_pizza: Identify the highest-priced pizza.
- 04_most_common_pizza_size: Identify the most common pizza size ordered.
- 05_top5_pizza_types: List the top 5 most ordered pizza types along with their quantities.

📈 Intermediate Analysis
- 06_category_wise_quantity: Join the necessary tables to find the total quantity of each pizza category ordered.
- 07_orders_by_hour: Determine the distribution of orders by hour of the day.
- 08_category_wise_distribution: Join relevant tables to find the category-wise distribution of pizzas.
- 09_avg_pizzas_per_day: Group the orders by date and calculate the average number of pizzas ordered per day.
- 10_top3_revenue_pizzas: Determine the top 3 most ordered pizza types based on revenue.

🔥 Advanced Data Analytics
- 11_pizza_revenue_percentage: Calculate the percentage contribution of each pizza type to total revenue.
- 12_cumulative_revenue: Analyze the cumulative revenue generated over time.
- 13_top3_pizzas_per_category: Determine the top 3 most ordered pizza types based on revenue for each pizza category.
